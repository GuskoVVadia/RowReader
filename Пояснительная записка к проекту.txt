Пояснительная записка

Версия Java 1.8
Графическая платформа JavaFX
Проект собран на Maven.
Приложение построено по схеме MVC.

Исполняемый файл собранного проекта находится в папке target проекта, под названием RowReader-jar-with-dependencies.jar
При запуске jar на исполнение ему передаётся имя файла или путь к файлу с указанием имени файла.
По алгоритму, после запуска приложения, элемент отвечающий за поиск пути (за это отвечает класс PathFinder) берёт абсолютный путь, где располагается исполняемый файл и в папке, в которой расположен проект ожидает увидеть файл (если при запуске было передано имя файла); или выстраивает новый путь, который был указан при запуске с именем файла. В обоих случаях происходит проверка на наличие файла по указанному или выстроенному пути.
В случае неудачи - через контроллер передаётся сообщение по интерфейсу AlertMassage с кодом ошибки, пояснением к ошибке по возможности, и дальше происходит вывод окна об ошибке.
В противном случае - запускается класс TextDetector, который определяет кодировку файла.
В случае неудачного определения кодировки - бросается ошибка - которая передаётся по интерфейсу AlertMassage с кодом ошибки, пояснением к ошибке по возможности, и дальше происходит вывод окна об ошибке.
В случае опеределения кодировки файла - запускается графический режим, с передачей в контроллер пути к файлу и кодировки файла.

Main - стартовый класс приложения, стартовая точка запуска графического приложения JavaFX.
Именно в классе Main происходит поиск пути к файлу и определение кодировки, путём запуска соответсвующих классов.
Класс Main собирает Stage и загружает Controller приложения. Загрузчик графического приложения ищет файл sample.fxml, в котором описаны элементы, которые будут участвовать в построении окна. 
Класс Controller реализует два интерфейса: AlertMessage (для формирования окна ошибки с выводом самой ошибки) и ObservableSize (для оповещения слушателей размеров окна с последующей передачей им размеров окна при изменении). Controller создаёт экземпляр Model и View, а также запускает слушателя размеров окна с задержкой и слушателя клавиатуры, для отлова нажатия на кпопки построчного или постраничного листания. При возникновении событий (нажата кнопка, изменился размер) Controller оповещает View по размерам окна, либо воздействует на соответствующие методы View.

Интерфейс View - расширяет интерфейс ObserverSize (слушателя окна) и описывает методы, какими должен владеть класс его реализующий.
Класс Page - реализует интерфейс View - является сущностью Вид, отвечает за вывод данных в окно приложения. Взаимодействует с Моделью данных, выводит текст в окно и реагирует соответсвующим образом на события, происходящие в Controller.
Интерфейс Model - описывает требования к классу, а именно выдача строки из Модели данных по требованию и максимальное число строк, имеющихся в наличии у Модели.
Класс FileReadingModel - реализует интерфейс Model. Конструктор класса запрашивает путь к файлу и кодировку файла, с которого будет производиться чтение строк в блок строк. Чтение из файла выбран как вариант менее затратный для работы с оперативной памятью, по сравненению с другими возможными моделями. Также выделяется своей простотой при дальнейшей работе приложения. При невозможности чтения из файла - выкидывается ошибка, которая передаётся через интерфейс AlertMessage (для формирования окна ошибки с выводом самой ошибки) и последующим выходом из приложения.

Класc ObserverSizeRow - класс в котором инкапсулирована строка из Model, с той лишь разницей, что при оповещении объекта строка начинает перестраиваться по указанной ширине окна. Новое, перестроенное значение, отдельно сохраняется в отдельную сущность, хранящуюся в объекте. 
Класс RowStorage - который производит из Модели набор блока строк, хранит их у себя и отдаёт по требованию, также по требованию очищает своё хранилище или набирает его, начиная с указанной строки.
Класс Word - как основная единица отображения. Класс хранит в себе слово, размер шрифта, которым его нужно написать/отобразить и габариты прямоугольника, в который это слово можно вписать системным шрифтов, указанного размера.
